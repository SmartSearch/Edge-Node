Replay Feed

Description: Replay a previous feed

Usage: Send an HTTP GET request to http://servername/CommonInterface/replayFeed?action=<selected_action> with the required action.

selected_action can be one of:
list: List the feeds being currently replayed
start: Start a new replay
stop: Stop a currently running replay
delete: Stops a replay if currently running and deletes it from database
delete_all: Stops all currently running replays and deletes all relevant feeds from database

The start, stop and delete actions require the feedname additional parameter with the name of the relevant feed

Other parameters:

start

newname : This optional parameter can be used to set the name of the replayed feed. Note that the name will be prefixed by "replay_"
repeats : This optional parameter specifies the number of repetitions (default 1). You can set it to <=0 for infinite repetitions.

If you want to specify a subsection of the feed to repeat, you can use the following optional parameters, which should be either in xsd:dateTime format (e.g. 2012-11-26T01:24:59.439Z) or in milliseconds since 1 January 1970 00:00:00 (e.g. 1353893099439).
start_date : Starting date/time
end_date : Finishing date/time

delete_all

confirm : this should be set to 'delete+all+replays' in order to avoid accidental deletions

Examples:

Single replay of the feed named 'aitathens' as 'replay_aitathens' (autogenerated)
http://servername/CommonInterface/replayFeed?action=start&feedname=aitathens

Replay a 2 minutes long section of the 'camerafeed' as 'replay_test_camera' in infinite loop
http://servername/CommonInterface/replayFeed?action=start&feedname=camerafeed&newname=test_camera&start_date=2012-11-06T12:47:36.500Z&end_date=2012-11-06T12:49:36.486Z&repeats=0

List currently running replays
http://servername/CommonInterface/replayFeed?action=list

Stop and delete ALL running replays
http://servername/CommonInterface/replayFeed?action=delete_all&confirm=delete+all+replays
